# Working agreements

- Read `README.md` and `Prompt.md` before making changes.
- Follow milestone order unless blocked.
- After each milestone, run the smallest relevant validation and update `README.md` or a docs note.
- Prefer simple, testable code over clever abstractions.
- Do not hardcode dataset paths or field names when configurability is feasible.
- Record unresolved assumptions in one central TODO section.
- Avoid heavyweight dependencies unless they are clearly necessary.
- Keep scripts runnable from the command line with `--help`.
- After changing data pipeline code, run the closest relevant tests first.
